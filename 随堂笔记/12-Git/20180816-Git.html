一,版本控制系统 1,定义:是记录一个或者若干个文件变化的软件; 分类: 1)本地版本控制系统; 记录文件变化,给文件打补丁的方式; 缺点:无法进行多人协作; eg: webstrom中有个VCS的,这是本地的版本控制系统; 可以管理编辑的网页源代码,PSD/PNG制图等所有文件; 2)集中式版本控制系统; 特点:有一台中央服务器,专门用于管理所有代码; 所有人通过上传和下载进行代码同步; 缺点:特别依赖网络,特别依赖远程中央服务器; eg: SVN; 3)分布式版本控制系统; 远程仓库,专门用于管理代码; 从远程仓库克隆一份到本地仓库(完整镜像); 本地写好代码后提交到本地仓库 本地仓库提交到远程仓库; 特点:有一个远程仓库,专门用于管理代码; 所有人本地都有一个远程仓库的完整镜像;每一次的提取操作,都是对代码仓库的完整备份; 优点:不是特别依赖于网络;不是特别依赖于远程仓库; eg: Git; 二,Git相关 1,安装;不要有中文目录;建议安装在c盘; 2, Git GUI 图形化的操作; Git Bash 命令行的操作;学习这个 3,登录github网址: www.github.com ; Sign in 登录 Sign up 加入 4,入门;属于分布式版本控制系统,用于敏捷高效的管理系统 5,简介:linux之父;早期用的bitKeeper,后来要收费了,linux之父linus Torvalds就开发了git; 6,初体验: git init ;初始化git仓库;创建一个隐藏文件夹(.git); 三个区: 工作区;暂存区;仓库区; 工作区:写代码的地方; 暂存区;仓库区; /git init创建的 .git文件;//暂存区是防止误操作的; 7,基本命令 1) git init 初始化仓库,创建一个隐藏文件夹(.git);包含暂存区和仓库区; 2) git status 查看文件状态; 红色 文件在工作区,即 Untracked files 未追踪的文件;modified修改的文件; 绿色 文件在暂存区,即 Changes to be committed有一些新变化,可以提交给仓库; (1) git status -s 简化输出文件状态; 3) git add 文件名; 将工作区的文件添加到暂存区; (1)提交当前目录下的所有文件,从工作区到暂存区: git add . git add -A git add --all (2)提交当前目录下css文件夹下面的所有文件到暂存区: git add css (3)提交当前目录下所有以js文件后缀名的文件提交到暂存区: git add *.js 4) git commit -m '提交信息';将暂存区的文件提交至本地仓库区//提示信息,对当前提交至仓库的文件备注一下信息; (1) 如果没有加提示信息,即 git commit ; 会进入vi编辑器,是Linux里面的纯命令行操作编辑器; 提示:空的提交信息会阻止提交; 强制退出 :q!; 重新提交即可; 会出现提示信息: 提交失败,被中止了; (2) git commit -a -m '提交信息'; 快速提交,从工作区至仓库区;//底层还是从工作区提交至暂存区;再从暂存区提交至仓库区; 缺点:无法提交未追踪的文件,即从来没有被git管理过的文件,Untracked files 未追踪的文件; (3) git --amend -m '修改的提示信息'; 只能修改最近一次的提交内容; 5) git log 查看提交日志; 日志中: commit后面是提交的版本号; (1) git log --oneline 将提交日志在一行显示; 6) git config --list 查看所有配置 7) git config --unset --global user.email 将全局指定的配置文件中user.email重置;user.name,可以重置用户名 没有配置邮箱和用户名(会提示 please tell me who you are) 8) git config --global user.email '邮箱'; 配置全局的邮箱和用户名; git config --global user.name '名字'; 8,进阶命令 1) git diff 查看每次提交的内容的不同; 如果添加的,命令结束后是++ 绿色的;如果是删除的,是-- 红色的; (1) git diff 比较工作区和暂存区的不同; (2) git diff --cached 比较暂存区和仓库区的不同; (3) git diff HEAD 比较工作区和仓库区的不同; (4) git diff 版本号1 版本号2 比较两个版本之间的不同;//一般复制前面的6-7位即可; 2) git reset 版本回退; (1) git reset --hard 版本号 重置工作区,暂存区,仓库区;回退到指定版本号;//版本号写6-7位即可; (2) git reset --hard head~0 当前提交;//~2上上次提交;~1上一次提交; (3) git reset --soft 版本号 只重置仓库区; (4) git reset --mixed 版本号 重置暂存区和仓库区; 3) git reflog 查看所有日志;//查看进行的所有操作; 4) 忽视文件; 如果文件已经被git管理了,用这个方法没有用,需要在项目一开始的时候就做好忽视的文件列表; (1)文件名固定: .gitignore ;//必须是这个文件名;windows用.gitignore.即可生成,或者在编辑器中生成.gitignore即可; (2)将需要忽视的文件名/路径/文件夹名放在这个忽视文件中,即可; 5)分支命令 分支: 多任务并行开发;分支之间互不干扰; 在进行了第一次提交之后会在仓库生成一个新的版本号;并且让一个master主分支指向这个新的版本; 分支其实就是指针; head指针,永远指向当前所在分支; 任何一次提交到仓库区,都会在仓库区生成一个新的版本 (1) giti branch 查看分支; 如果分支是绿色的,并且前面有朵白色的小菊花(*);说明是当前分支; 如果分支是白色的就是其他分支; (2) giti branch 分支名 创建分支; (3) giti checkout 分支名 切换到指定分支; giti checkout -b 分支名 创建并切换分支; (4) giti merge 分支名 合并分支;在当前分支下,合并指定的分支进来到当前分支;将其他分支的内容合并到当前分支 (5) geiti branch -d 分支名 删除分支;当前是自己的分支下,不能删除自己,需要切换到其他分支,再删除分支; 注意:在开发一些大型功能时,都会新开一个分支,先在分支上进行开发,开发完成后,一次性合并到主分支; 合并分两种情况: (1)如果当前分支 是合并分支的祖辈分支,会进行快速合并; (2)如果不是祖辈分支,可能会产生冲突(两个分支修改了同一个文件) 分支冲突: 冲突现象: 会在冲突文件中出现提示标记 <<<>>>>>分支名 冲突提示:会在路径后面出现后缀: master|MERGE 解决方法: (1)手动把文件内的提示标记删掉; (2)然后再提交; 9,远程仓库操作命令 git和github没有直接关系; git是版本控制工具; github是代码托管平台,开源社区,是git的一个远程代码仓库; 1)git clone 远程仓库地址 文件夹名 从远程仓库克隆一份到本地仓库,放在指定文件夹下;//完整镜像 远程仓库的地址是在github上复制下来的; 有两种: HTTPS;每次都要输入用户名和密码; SSH;可以配置免密登录; 2) git push 远程仓库地址 分支名 将本地仓库的内容放到远程仓库去;通常先pull一下再push; 先把远程仓库的代码pull到本地,再把自己的push到远程代码上去; git push origin 分支名 进行克隆的时候,已经把地址和变量名(origin)进行关联起来了;后面可以直接用; 3) git pull 将远程仓库改变过的内容镜像到本地仓库; 10, 变量 1)git remote 查看所有变量; 2) git remote add 变量名 仓库地址; 3) git remote remove 仓库别名; 移除变量名 11,SSH免密登录 配置: 注意：这些命令需要在bash中敲 - 1 创建SSH Key：ssh-keygen -t rsa //然后一路回车 - 2 在文件路径 C:\用户\当前用户名\ 找到 .ssh 文件夹 - 3 文件夹中有两个文件： - 私钥：id_rsa - 公钥：id_rsa.pub - 4 在 github -> settings -> SSH and GPG keys页面中，新创建SSH key - 5 粘贴 公钥 id_rsa.pub 内容到对应文本框中 //在远程仓库中,找到SHH and GPG keys ;把公钥粘贴到key中;
